About this tool

SEP Tracker is not an official City of Greensboro product. It is an independently built civic transparency tool, built by Andrew Randall / ARXI Systems from publicly available City documents and data, with no official affiliation with the City of Greensboro, the Office of Sustainability and Resilience, or the Community Sustainability Council.

It exists to put the City's Strategic Energy Plan (2022) and its Annual Progress Reports side by side with independently verified public-record facts (Legistar, press releases, live City data), so residents, journalists, City Council, and CSC members can see a claim's evidentiary weight at a glance rather than take every number on faith.

How to read a citation

Every figure and status badge on this site carries a small colored tag you can hover or tap:

  • ①SEP '22 — stated in the 2022 Strategic Energy Plan itself.
  • ②Progress '25 — stated in the City's own 2025 Annual Progress Report.
  • ③Verified — found independently (Legistar, press coverage, etc.) and confirmed against full primary-source text.
  • ④Unconfirmed — found independently, but resting only on a search-result summary because the primary document couldn't be fetched. Treat as provisional.

Tiers 1–2 are the City's own reporting. Tiers 3–4 are this tool's own research, layered on top — and deliberately never rendered to look like the City said them.

Why independent research is in scope

The City's own two reports are not a complete record of its renewable-energy activity — they're what OSR chose to compile and publish. This tool surfaces true, independently-verified facts the City's own reporting omitted (a federal transit grant explicitly credited to this plan; a City-owned solar array never mentioned in either progress report) because that's part of an accountability tool's job, not scope creep. It's held to a higher bar for doing so: Tier 3/4 badges, explicit "single-source-unconfirmed" flags, and documented correction notes where earlier research turned out to be wrong (see data/SCHEMA.md).

Data gaps are shown, not hidden

Where the City's own reports don't address something — a goal with no 2025 status update, a figure the report itself says was unavailable — this tool shows that gap explicitly rather than omitting the row. A distinction is drawn between an unexplained silence ("Not Reported") and a documented deprioritization the SEP itself explains ("Deprioritized") — they are not the same finding and shouldn't read the same.

Live external data sources (in progress)

Beyond the two static source documents, this tool is starting to pull live data directly from City systems, verified working rather than assumed:

  • NREL PVWatts — a live "estimate solar potential" calculator on the White Street Landfill and Buildings (Engineering & Inspections) pages, calling NREL's PVWatts v8 API directly from your browser. Confirmed working with real output during testing (verified 2026-07). Worth flagging: NREL's developer API migrated domains as of a May 2026 reorganization — the lab is now branded "National Laboratory of the Rockies" (NLR) and the API now lives at developer.nlr.gov rather than the old developer.nrel.gov, which no longer resolves.
  • Canopy Hub (canopy.greensboro-nc.gov) — the City's Esri Hub site for the tree canopy/equity prioritization project. Its Search API is live and returns real GIS layers, including CEJST_Districts2024 (a federal environmental-justice districts layer) and Neighborhoods_Point — both real FeatureServer endpoints, confirmed live below:
    Checking live…
  • Open Gate City (data.greensboro-nc.gov) — confirmed live, 79 datasets, including Code Compliance case history, Fire incidents, and Police crime data via the same Esri Hub Search API pattern. Not yet wired into a specific page.
  • Gate City Budget (budget.greensboro-nc.gov) — checked, but its Search API returned the same generic city-wide catalog as Open Gate City above, not a distinct CIP/budget-line-item dataset. No Capital Improvements Program data was found at this endpoint to cross-reference against the energy-cost figures in baseline-2019.json. Flagged as an open question rather than a working integration — City budget/CIP data may live in a separate system (e.g. a dedicated OpenGov/Socrata budget explorer) not yet identified.
  • EIA grid-mix data for Duke Energy Carolinas — infrastructure exists (.github/workflows/refresh-grid-mix.yml, a scheduled GitHub Action) but is not yet running: it needs a free EIA API key added as a repository secret. Until then, data/live/grid-mix.json does not exist.
